# Telegram-бот (`src/app/bot/`)

## Назначение

Бот объединяет два независимых сценария учёта заправок (см. ТЗ):

1. Операции по **топливной карте** — данные приходят из импорта API (код импорта и клиент API в репозитории не менялись в рамках этой задачи).
2. Операции **за личные средства** — пользователь присылает фото чека; дальше OCR, проверки по БД и строка в Excel.

## Первый запуск и приветствие

- При **первом** `/start` для данного Telegram ID показывается текст (и при наличии файла — **изображение**), где кратко объясняется, зачем бот и как им пользоваться.
- Список уже показанных ID хранится в `exports/welcome_shown.json` (создаётся автоматически).
- Путь к картинке:
  - переменная окружения `WELCOME_BANNER_PATH` (абсолютный путь к файлу), **или**
  - файл по умолчанию: `src/app/bot/assets/welcome.png` (положите сюда свой баннер).

## Команды и кнопки (пользователь)

| Действие | Описание |
|----------|----------|
| `/start` | Меню, приветствие (один раз), возврат в главное меню |
| `/link <код>` | Привязка Telegram к учётной записи |
| `/check`, «📸 Отправить чек» | Старт сценария личных средств |
| `/myprofile`, «👤 Мой профиль» | ФИО, карты, авто в профиле |
| `/pending`, «⏳ Мои уведомления» | Ожидающие подтверждения операции по карте |
| `/help`, «❓ Помощь» | Краткая справка |

Регистрация хендлеров: `src/app/bot/handlers/user.py` → `register_user_handlers`.

## Где искать логику

| Область | Файл |
|---------|------|
| Пользовательские сценарии, FSM | `handlers/user.py` |
| Inline/reply-клавиатуры | `keyboards.py` |
| Уведомления по операциям с карты | `notifications.py` |
| Права и активность | `permissions.py`, middleware в `register.py` |
