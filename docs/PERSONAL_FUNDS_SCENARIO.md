# Сценарий «Заправки за личные средства»

Соответствие **ТЗ, раздел 8.2** и коду.

| Шаг ТЗ | Поведение в боте |
|--------|------------------|
| 1. Фото чека | Кнопка «📸 Отправить чек» или `/check` → состояние `waiting_for_photo`, пользователь шлёт фото. |
| 2. OCR | `SmartFuelOCR.run_pipeline()` в `src/ocr/engine.py`: Tesseract + LLM-структурирование по схеме `ReceiptData`. |
| 3. Реквизиты | Запись черновика в `fuel_operations` (`source=personal_receipt`, `ocr_data`, `doc_number`, `date_time`, `presumed_user_id`). |
| 4. Показ данных | Сообщение с топливом, литрами, суммой, АЗС, номером чека, датой/временем. |
| 5. Подтверждение | Inline: **Подтвердить** / **Исправить** / **Отменить** (`get_ocr_confirm_kb`). |
| 6. Госномер | После «Подтвердить» — ввод текста (`waiting_for_personal_car_plate`). |
| 7. Проверка авто | Нормализация через `plate_util.normalize_plate`, поиск в таблице `cars` (`find_cars_by_normalized_plate`). Нет в справочнике — повторный ввод; несколько строк — выбор кнопками (`personal_car_*`). |
| 8. Кто заправлялся | Запрос ФИО (`waiting_for_personal_fueler`). |
| 9. Проверка человека | Поиск активного пользователя `users` по `full_name ILIKE`; 0 / несколько — уточнение. |
| 10. Excel | `export_to_excel_final(op_id)` → лист `Заправки_личные_средства` в `exports/Fuel_Report_Master.xlsx`. |

## Кнопки ТЗ 9.4

- **Подтвердить** — переход к госномеру.
- **Исправить** — выбор: **ввести данные вручную** (шаблон текстом в одном сообщении) или **новое фото** (черновик удаляется, снова `waiting_for_photo`).
- **Отменить** — удаление черновика и выход из сценария.

## Безопасность черновика

Колбэки OCR доступны только инициатору: совпадение `presumed_user_id` с пользователем, статус операции `new`, источник `personal_receipt`.

## Связанные модули

- `src/app/bot/handlers/user.py` — FSM и хендлеры.
- `src/ocr/engine.py` — пайплайн OCR.
- `src/app/excel_export.py` — колонки и листы Excel.
- `src/app/plate_util.py` — госномера.
